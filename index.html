Cut LLM context by 65–94%.

Gist is a high-performance Go MCP server that sits between agentic clients (Claude Code, OpenCode, Aider) and your filesystem to prune, restructure, and budget the information sent to the model.

Install Read the docs Source
$go install github.com/elbader17/gist/cmd/gist@latest

The problem

Modern agentic coding tools have a new bottleneck: unnecessary context saturation.

📖

Whole files

Agents read entire files of thousands of lines when signatures and types would suffice.

📢

Massive build logs

Multi-megabyte compiler output injected into every prompt.

🔄

Runaway loops

Agents retry the same failing test indefinitely, burning budget with no progress.

Six tools. One goal: spend fewer tokens.

AST-aware pruning

Go files collapse via go/parser + go/printer. Multi-language signature-only pruning for Python, JS, TS, Rust, Java, C/C++, Ruby.

LRU prune cache

Keyed by (path, mtime, size). Repeat reads return <1ms with a cache_hit flag.

One-call optimizer

squeeze_context prunes, aligns, and caps tokens in a single round-trip. Returns combined prompt + savings metrics.

Cache-friendly markdown

Aligned output uses stable per-layer hashes so provider-side prompt caches reuse the prefix across calls.

Parallel diff enrichment

Worker pool (up to 8 goroutines) replaces serial per-file git diff. ~2x faster on multi-file changes.

Loop circuit breaker

Detects repeated actions, cost ceilings, and token ceilings. Sessions persist via a 2s debounced flusher.

MCP Tools

Registered automatically on stdio JSON-RPC. Configure your client once.

view_file_slim cache-aware

Prune a file to signatures. Go files via AST; other languages via signature-only extraction.

enforce_budget circuit breaker

Track session cost/tokens; halt runaway loops before they burn the budget.

align_context_cache cache

Reorder prompt components into 4 layers and return markdown with stable hashes.

fetch_diff_context parallel

Replace raw git diff with semantic summaries and log/comment-only detection.

squeeze_context flagship

One-call optimizer: prune + align + cap. Returns a single ready-to-send prompt and savings metrics.

report_savings telemetry

Cumulative token-savings telemetry per session and per tool, persisted across restarts.

Install & run

As MCP server

{
  "mcpServers": {
    "gist": {
      "command": "/usr/local/bin/gist",
      "args": []
    }
  }
}

Wrap any command

$ gist wrap -- claude
$ gist wrap -- aider

# every prompt captured, aligned, and persisted to
# ~/.config/gist/captures/<ts>-<pid>.jsonl

# shell alias for permanent capture:
alias claude='gist wrap -- claude'

Architecture

Single static binary. Zero dependencies. CGO_ENABLED=0.

Core

  • pkg/ast — Go AST + multi-language pruner
  • pkg/aligner — 4-layer reorder + markdown
  • pkg/tokenizer — BPE-style counting

Persistence

  • pkg/budget — sessions + debounced flush
  • pkg/metrics — savings telemetry
  • pkg/config — JSON config

Efficiency

  • pkg/cache — LRU prune cache
  • pkg/squeeze — one-call optimizer
  • pkg/diff — parallel enrich
